Detailed Description

Returned by pdf_stream::open_for_read() for reading from a PDF stream.

This is a legacy class. There are more efficient and effective ways to read a t4::data_block_stream. Potential clients might want to consider just calling t4::data_block_stream methods directly.

void t4::pdf::stream_reader::read_bytes ( vector< unsigned char > &  buffer,
data_store_size  bytes_to_read 
)

Reads binary data from the stream and puts the data into the specified buffer starting at position 0.

Reads the number of bytes specified in bytes_to_read, and will block if necessary until either that many bytes are available or end-of-stream is reached. Upon exit the value of buffer.size() will hold the number of bytes actually read. This will always match bytes_to_read, except possibly for the final chunk of data in the stream. As a special case, the value 0 means to read the entire stream.

The file position is remembered, and the next call to this method will start reading again from that position.

It is harmless to continue calling this method after all data has been read. No data will actually be read and buffer.size() will be set to 0.

Behavior is undefined if the backing PDF stream is modified while this instance is in existence.

Parameters
bufferBuffer where the data will be read into.
bytes_to_readNumber of bytes to read.
